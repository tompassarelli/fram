## NEW files

### `bin/fram-ingest-code` (NEW, executable)
**Stage 1 — INGEST.** `.bclj` tree → `.fram/code.log`. For each module: `racket --emit-edn`,
re-key each file-local integer id `n` to the node name `@<module>#n` (subject int → `:l
"@mod#n"`; integer object → `:r "@mod#o"` link; string object → `:r` literal), plus one
`@<mod>#root file "<path>"` bookkeeping claim. Appends flat-log lines the EXISTING daemon
boots from unchanged (`serve-flat <PORT> <code.log>` → `migrate-flat->co`). `--module
<name>` restricts to one module (the flip experiment passes `--module schema`). The log
carries ONLY the AST (kind/v/fN/child/segN/commentN/tail/style/placement); `refers_to` +
render markers are DERIVED, never persisted (verified: 0 `refers_to` lines in the log).

### `bin/fram-render-code` (NEW, executable)
**Stage 2 — RENDER-FROM-LOG.** `.fram/code.log` → `.bclj` (a pure function of the log).
IN-PROCESS `migrate-flat->co` over the log (no socket — a pure render needs no
coordinator), `resolve/resolve-warm-store!` materializes `refers_to`, `resolve/extract-file!`
projects the module's claims to resolved EDN, `racket --render` → `.bclj`. `--log` /
`--out` selectable (default `$PWD/.fram/code.log` / STDOUT).

### `bin/fram-commit-code` (NEW, executable)
**Stage 3 — COMMIT-THE-DELTA** (the heart of the flip). Given the verb's recompile-gated
new `.bclj` + the module + `--port`, it commits the AST delta to the code log THROUGH the
coordinator's single-(te,p,r) `:assert`/`:retract` wire at base version (retry on
`:conflict`, mirroring `fram_mcp.clj route-write`). **SAFETY GATE:** refuses unless the
coordinator's `:status :log` basename contains `code.log` (never commits code to a
thread/lodestar log). **Algorithm = WHOLE-NODE clean-slate re-commit, NOT an id-keyed
set-difference** — see "Key finding" below. Restricts to AST predicates only (a `relates_to`
bridge or `@<mod>#root file` claim on a code node is never churned by a code edit).

### `bin/fram-render-code-all` (NEW, executable)
**Stage 4 — GENERATED-MARKER build step.** Regenerates every `;; @claim-canonical`
sentinel'd module from the log (only `schema.bclj` is adopted today). **DRY-RUN by default**

`#lang beagle/clj` → `(define-target clj)` and the committed `schema.bclj` still carries

## KEY FINDINGS (the falsifiable normalization + a correctness fix — read these)

### 1. The KEYSTONE byte-identity is vs **render(text)**, not the hand-committed bytes.
`racket --render` is a normalizing pretty-printer, NOT a byte-preserving copier. Verified:
`render(committed schema.bclj's resolved EDN)` is **5673 bytes**, the committed file is
**6006 bytes** (differ by 333 B). One of the byte differences is a true normalization; the
other is a **DELETION**, not a reflow:
- `#lang beagle/clj` → the leading `(define-target clj)` form (a reader directive the
  renderer canonicalizes; it round-trips datum-identically + re-reads identically).
- **COMMENT DELETION (NOT "whitespace reflow") — the flip is LOSSY on inline/in-body comments.**
  4 comments present in committed `schema.bclj` are absent from render(log), render(text),
  AND `.fram/code.log`: `; RESERVED: claim-level supersession` (schema.bclj:41),
  `; trigger; distinct from any domain` (:42), and the 2-line in-body block
  `;; identity lives in the graph ...` / `;; those don't survive dump/replay ...` (:45-46).
  The loss originates at `racket --emit-edn` (the lexer drops trailing-inline + in-body
  comments) so they never enter the log. `render(log)==render(text)` passes only because BOTH
  sides drop the same comments. Keystone scope is therefore **program-semantics-identical;
  source comments NOT preserved** — DESIGN §8 risk #6 ("a real loss masked as normalization")
  is realized here. `bin/fram-render-code-all --write` is currently UNGUARDED against this loss.

The `;; @claim-canonical` sentinel block **round-trips NATIVELY** as AST comment/seg nodes
(verified — no re-prepend needed; DESIGN §3 "the gate DETERMINES which" → native). So the
honest, falsifiable keystone is **`render(log) == render(text)` byte-identical** (the SAME
projection the existing authoring edge produces and trusts), which is **PASS**, PLUS
semantic compile-identity: `render(log)` compiles to the committed `out/fram/schema.clj`
modulo only `^{:line N :file ...}` source-position metadata (which is a function of the file
path, not the AST). Any byte difference NOT covered by this stated normalization FAILS the
gate. **This is narrower than "cmp -s vs the committed .bclj" the DESIGN's prose suggested —
the reviewer should treat render(log)==render(text) as the load-bearing claim.**

### 2. The delta-commit is **WHOLE-NODE clean-slate**, NOT an id-keyed set-difference.
`emit-edn` assigns node ids by deterministic traversal, so a NON-trivial edit **renumbers
every node after the edit point**. An id-keyed set-difference (DESIGN §4.1's literal
"adds = POST\\PRE, removes = PRE\\POST") then leaves coincidentally-shared ids (e.g.
`@schema#100` is a DIFFERENT logical node pre vs post) carrying a MIX of old + new claims →
a CORRUPT AST (a one-line body change measured a 3204+3325 spurious "delta", and an early
build produced a `--render` crash: `hash-ref: no value found for key "placement"`).

`bin/fram-commit-code` therefore commits a **per-node clean slate**: for every node whose
full claim set differs pre↔post, retract ALL its current claims, then assert its full new
claim set (kind/v leaves before fN/seg/comment/tail edges, so a child exists before a parent
points at it). Postcondition is exactly `live AST == emit-edn(new)` regardless of id
alignment — total, idempotent on the unchanged majority. **Cost: a non-trivial edit
re-commits ~the whole module (~1172 nodes / ~8000 log lines for schema).** This is correct
ON SUCCESSFUL COMPLETION + append-only-safe, but it is NOT the surgical delta the DESIGN
envisioned. A truly surgical delta needs id-STABLE node identity, which the text-roundtrip
`emit-edn` path does not provide (the resolver's OWN mint/supersede assigns stable ids
internally — committing the resolver's internal ops, rather than an emit-edn re-diff, is the
path to a surgical delta; out of scope here, noted as FOLLOW-UP).

> **CORRECTION — does NOT close DESIGN risk #2 (B2): the whole-node re-commit is NON-ATOMIC.**
> `bin/fram-commit-code:189` runs ALL retracts first (`doseq … :retract`), THEN all asserts
> (`:195`); each `commit-op!` (:125-132) is an independent coordinator round-trip that `die`s
> (System/exit 1) on a non-conflict reject / socket failure / daemon death. The wire has NO
> batch/tx op. A mid-sequence abort AFTER the retract phase leaves a node MISSING required
> claims — render then fails with the exact `hash-ref: no value found for key "placement"`
> crash documented above (which is a TORN-COMMIT failure, not only the id-aligned-mix one).
> So the per-node clean-slate is total only if the whole op-stream completes; a torn commit
> can leave the code log durably MISSING required claims (it LOSES, not only GAINS). The
> `.bclj` source is the recovery oracle until full adoption. This is a KNOWN BLOCKER for
> default-on — see DESIGN §"KNOWN BLOCKERS" / risk #2.

> **NOTE — idempotency depends on AST preds being MULTI-VALUED (m8).** The whole-node
> re-commit is idempotent/re-runnable only because the AST preds (`child/kind/v/tail/style/
> placement`, the bare-name preds) are multi-valued. `out/fram/kernel.clj:4` reads
> `FRAM_SINGLE_VALUED` ONCE at ns-load (env-frozen); a boot under e.g.
> `FRAM_SINGLE_VALUED="child …"` would make `child` single-valued, collapsing a multi-child
> node to one child on re-commit → corrupt AST. (`fN/segN` are immune — `ck/single?` is
> exact-string.) This NEVER fires under standard boot (default + lodestar vocab are disjoint
> from AST preds), but it is env-overridable and ungated — no gate asserts the code
> coordinator's effective cardinality.

### 3. The AST predicate vocabulary is `kind/v/child/tail/style/placement` + `fN/segN/commentN`.
`style`/`placement` (on comment nodes) are easy to miss — omitting any emit-edn pred from
`ast-pred?` drops it on the whole-node re-commit and makes `--render` throw. The set in
`bin/fram-commit-code` is the COMPLETE emit-edn vocabulary (verified against the corpus).
