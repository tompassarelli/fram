# Reasoning cost: querying the graph vs reconstructing structure from text

An agent answering a **structural question** about a codebase ("who calls this
cross-module def?", "what is the ultimate definition behind this alias?", "what is
the transitive blast radius of changing X?", "which functions take `(Vec Claim)`?")
can do it two ways. It can **query a live claim graph** that already holds the
canonical structure — `refers_to` identity, scope-correct callers, reaches-closure,
resolved through re-exports/aliases, pre-paid once and kept fresh incrementally
(S1/S3). Or it can **reconstruct that structure from text** — grep the source,
re-read files, learn each module's aliases, disambiguate spellings, chase the alias
to its definition, and backtrack when a grep over-matches. The graph holds the
structure; text holds none of it, so the text agent rebuilds it every time.

This experiment measures the **reconstruction work** — the count of retrieval
operations to reach the *same verified-correct answer* — across a task suite that
grows in structural distance (hop-depth). It is **tier-2, the reasoning-cost axis**.

> **The headline metric is op-count, not wall-clock — and the two evidence layers
> answer two different questions.** Layer A is a **structural lower-bound PROOF**
> about *this corpus* (the minimal-correct text reconstruction, justified op-by-op,
> with no agent in the loop — so it is model-independent *because the agent is
> removed*, which also means it proves nothing about what an agent actually *does*).
> **Layer A is the empirical result of this experiment**, and it is regenerated by
> `reproduce.sh`. Layer B (real paired sub-agents) is a **DESIGN, NOT RUN** — it is
> specified below as future work but is not implemented and produces no measured
> result today. We do not present authored Layer-A numbers as if they were measured
> agent behavior, and we do not present the static `.behavioral/` artifacts as a
> Layer-B measurement. (This is must-fix #1 from the confound critique, made
> structural.)

## Read this first: what this is NOT

- **NOT a wall-clock benchmark.** "The graph arm finished in 40s, the text arm in
  90s" is the WRONG metric — confounded by model speed, task luck, network/JVM
  warmup, and serialization latency. We **do not** headline it. The model-independent
  metric is the **count of retrieval operations** to reach the correct answer — graph
  queries vs greps + file-reads + alias-chases + backtracks. "1 op vs 2 ops, same
  correct answer, everything else held constant" is the result. Op-count is
  insensitive to model *speed*. It is **not** insensitive to model *grep-skill* — a
  more skilled text agent picks a sharper regex and uses fewer ops; Layer A pins this
  by fixing the *minimal-correct* path, and Layer B measures the real spread (see
  *The metric* and *Layer B*).

- **NOT a wall-clock smuggle through the stopping rule.** Op-count stays
  model-speed-insensitive only if both arms run to the **same, substrate-blind
  stopping condition**. We therefore do **not** let sub-agents self-terminate on
  confidence ("I think that's it, done"). The stopping rule is identical and scored
  *after the fact*: **"stop when you can name the full answer set required by the
  prompt"** (the exact caller set / the single ultimate binding + its reference set /
  the `(Vec Claim)` fn set). A model that is calm on the graph arm and anxious on the
  text arm cannot inject a temperament confound, because neither arm decides its own
  stopping point on felt confidence. (Confound must-fix: stopping-rule.)

- **NOT a different axis from the sibling.** `../concurrent-authoring/` tests the
  **safety axis** — does text BREAK under concurrency. This tests the **cost axis** —
  does a *single* agent, no concurrency, do FAR LESS WORK to reach a structural answer
  through the graph. No concurrency is involved.

- **NOT "the graph is free", and the amortization frame is stated up front.** The
  graph PRE-PAYS the structural computation — project the AST to claims, resolve every
  reference to its binding identity (`refers_to`), maintain it incrementally on edits
  (S1/S3). That cost is real and is paid **once, at setup** (~11 `racket --emit-edn`
  invocations + a JVM daemon boot + a whole-corpus resolve walk; **measured: ~37s to
  a listening daemon, 101,671 live claims over the 11-module projection**). **The
  per-query advantage is a property of a WARM, already-running daemon — the explicit
  precondition of the per-query claim.** In a cold single-question session the graph
  loses (you pay the whole ~37s setup for one query); in a long-lived session (a
  deployed lodestar where the daemon is already up — the actual deployment) the setup
  is amortized to ~0/query and the per-query op-count is what "reasons faster in a
  context window" means. **We apply this frame to BOTH arms symmetrically** (must-fix:
  amortization asymmetry): the warm-session frame means the graph's setup is paid once
  *and* the text arm learns each module's alias once and reuses it across tasks (the
  alias-discovery read is amortized, NOT charged per task — see *The metric*).

- **NOT a strawman text baseline.** The text arm is the **minimal correct
  reconstruction a competent engineer would actually do with ripgrep** — including a
  **single-pass alternation regex** (`tsearch '(k/)?(->)?Claim\b'` is ONE search
  pass = ONE op; forbidding alternation would artificially serialize what one regex
  does and inflate the count — must-fix: allow alternation). We re-derived every
  text-minimal count *down* to this fair floor against the live corpus; several
  dropped (a: 4→2, b: 5→2, d-source: 4→3). The thesis must survive the FAIR (lower)
  text count — and it does. A skeptic attacking the text baseline as inflated must
  lose; a skeptic attacking it as a strawman must lose.

- **NOT a claim the graph "understands" code.** The graph transports identity and
  derives reaches-closure by Datalog; *correctness* of every answer is checked against
  ground truth computed once from the verified-correct resolver
  (`chartroom/src/resolve.clj`) and the recompile oracle. The graph never grades
  itself.

- **NOT a win on trivial-local lookups — and we CONCEDE it.** For a single-file local
  question, grep is one op and the graph is one query: a tie. We include a
  trivial-local calibration task (`cal`) and concede parity explicitly.

- **NOT a graph arm that secretly re-parses text.** The experiment-invalidating trap,
  audited in `../owned-resolution-audit/REPORT.md` (Q4, which scored a prior design
  `graph_arm_valid = false`). Every graph-arm retrieval is a **daemon socket
  round-trip against resolution-produced warm claims**. No graph-arm op shells
  `racket --emit-edn` or `bb resolve.clj callgraph`. The four guarantees are spelled
  out under *Owned-resolution validity* — **and all four are now verified to pass
  end-to-end** (the daemon boots over the projected corpus and serves nonempty
  `:callers`; see *The setup the graph pre-pays*).

## The phenomenon (the headline)

The same structural fact costs the graph **one query** and costs text a **sequence of
retrievals that grows with hop-depth**, because text must rebuild the structure the
graph already holds. The advantage is **monotone in hop-depth**: ~tie at 0 hops
(trivial-local) and **also a tie at the caller-MODULE-SET question** (direct callers:
1 vs 1 — text reads the module set off one search's line content; CONCEDED), **small
but real at 1 hop** (ultimate-through-aliases: 1 vs 2 — the clean live headline; text
must run an alternation pass AND prove the spellings converge on one binding), **large
at k hops** (transitive blast radius: 1 vs 9 — a frontier-batched BFS over the 7-member
closure), and **categorical** on the erased-type case (text-on-*lowered* cannot answer
the generic case at all).

All coordinates below are MEASURED against this corpus (`src/fram`, 11 modules — see
`tasks.edn` for the exact commands and `reproduce.sh` for the live re-measure):

- The record type `Claim` has **three live spellings** — `Claim` (39 occurrences,
  intra-kernel), `k/Claim` (29, cross-module annotations), `k/->Claim` (6, the
  generated constructor) — **= 74 tokens, ONE identity**. A *single* alternation regex
  `(k/)?(->)?Claim\b` catches all three in **one search pass** (verified: 39 + 6 + 29).
  The graph's `:callers` resolves all of them via `refers_to` + `ultimate` in **one
  query**, returning the **10 distinct `[module, spelling]` reference groups**
  (verified live), all proven to be the one binding.
- `6` modules alias `fram.kernel` as `:as k` (export, fold, import, main, query,
  tools); `datalog` and `schema` do **not**. In a **warm session** the text agent
  learns `k = fram.kernel` **once** and reuses it; we charge that alias-discovery read
  **once across the suite**, not per task (amortization symmetry).
- `thread-ids-i` (kernel.bclj:254) — the suffix `-i` is **distinctive**: a single
  `tsearch thread-ids-i` catches every reference INCLUDING the intra-kernel
  *unqualified* caller `work-thread-ids-i` (kernel.bclj:267). The substring trap is
  **one-directional** (verified): searching the *shorter* `thread-ids` over-matches
  `thread-ids-i`, but searching the *longer, distinctive* `thread-ids-i` is clean. So
  the text-minimal path is `tsearch thread-ids-i` + **one** read of the same `-n`
  output to drop the def line (:254), the comment (:265), and the self-name (:266),
  keeping the real call (:267) and the 5 cross-module call sites — **2 ops, correct**.
  The graph is **1**. (The *wrong-cheap* text path is real and worth stating: a
  qualified-only `k/thread-ids-i` grep MISSES the kernel caller → cheap but WRONG.)
- `41` `(Vec Claim)`/`(Vec k/Claim)` type-annotation **sites** exist in the **source**
  (verified: 20 `(Vec Claim)` + 21 `(Vec k/Claim)` = 41); they **erase to 0** in the
  lowered `.clj` (verified: `Vec Claim` appears 0 times in `out/fram`). This `41` is
  the **(Vec Claim) generic-parameter site count specifically** — distinct from the
  resolver's `116` total **type references across all types** (don't conflate: 41 ⊂
  116). A text agent on the lowered target is blind to the **generic** case; one on
  the source must grep + disambiguate all 41 by hand. The graph holds the type
  reference as identity regardless.

## The clean AB — what differs, what is held identical

**The ONLY thing that differs between arms is the retrieval substrate.** Everything
else is byte-identical.

| held IDENTICAL across both arms | the ONE difference |
|---|---|
| the corpus (`src/fram`, 11 modules, one frozen mktemp copy per run) | **text arm:** a logged grep/read CLI over the source files |
| the agent harness + model | **graph arm:** a logged query CLI over the live daemon |
| the task prompt (verbatim) and the required answer shape | (nothing else) |
| the starting context (the **learned-alias map**, given to BOTH arms once — warm-session frame) | |
| the substrate-blind stopping rule ("name the full answer set") | |
| the ground-truth scorer (same oracle for both) | |

- **Text arm CLI (`tsearch`/`tread`)** wraps exactly two primitives, each performing
  **one** retrieval: `tsearch <regex> [path]` (one search of the source — **alternation
  inside one regex is allowed and counts as one pass**) and `tread <file> [a:b]` (one
  file/range read). Nothing else — no daemon, no resolver, **no raw shell**. The arm
  reconstructs structure from these two primitives only.
- **Graph arm CLI (`gquery`)** wraps the daemon socket client (`(client port {:op …})`,
  `cnf_coord_daemon.clj:883`) and **nothing else**: `gquery :callers {…}`,
  `gquery :query {…}`, `gquery :status`. Each invocation is **one** socket round-trip
  against the warm graph. The wrapper is structurally incapable of shelling
  `racket`/`bb`/`resolve.clj` (it speaks only the daemon wire protocol).
- Both CLIs **append one line per retrieval to an arm-specific oplog**; the metric is a
  pure `wc -l` of that log.

The substrate is the independent variable. Op-count and correctness are the dependent
variables. There are no others.

## The setup the graph pre-pays (the one-time, visible cost — VERIFIED end-to-end)

Before any measurement, the harness pays the graph's pre-computation **once**, in the
open. This is the make-or-break step the fairness audit flagged as "unproven", and it
is now **proven to work**:

1. **Project the corpus to claims.** For each of the 11 `.bclj` modules, run
   `racket …/claims-roundtrip.rkt --emit-edn <mod.bclj>`. This yields the AST as bare
   `[id pred value]` triples with an `@file` header and **per-file integer ids**.
2. **Translate to the daemon's flat-log shape.** `migrate-flat->co` reads
   `fram.rt/read-log` format — **one EDN map per line `{:tx :op :l :p :r}`**
   (`cnf_coord_daemon.clj:922`, `src/fram/rt.clj:144-157`), where `:r` is a `ref-str?`
   (`"@…"` string) for edges and a quoted EDN string for literals. The harness's
   **projection translator** converts each module's `[id pred value]` vector into a
   `{:tx :op "assert" :l "@<module>#<id>" :p "<pred>" :r …}` map, **namespacing every
   node id by module** (`@kernel#1518`, `@main#…`) so the 11 files' integer ids do not
   collide, and quoting literals / ref-stringing edges. All 11 modules' lines are
   appended to **one temp flat log** (the daemon's canonical artifact for this run).
   *(Verified: the translator emits a 78,565-line flat log; the daemon folds it to
   101,671 live claims.)*
3. **Boot the daemon over that temp log** on a verified-free high port
   (`clojure -M cnf_coord_daemon.clj serve-flat <PORT> <TEMP-LOG>`). On boot,
   `migrate-flat->co` folds the log into the warm store (so the warm store holds the
   real corpus AST as claims, `:915`), and the first `:callers` triggers
   `ensure-refers!` → `resolve/resolve-warm-store!` (`:463`), which runs the lexical
   walk and **materializes `refers_to` over the warm store**. *(Verified: boot to
   listening ≈ 37s.)*
4. **Boot guard (fail-loud).** Before any task runs, `reproduce.sh` asserts the warm
   store is genuinely corpus-backed (see *Owned-resolution validity*, guarantee iv).
   *(Verified live: `{:op :status}` → `{:version 78565, :claims 101671, :log
   "<temp>/corpus.flatlog"}`; `{:op :callers {:module "kernel" :name "thread-ids-i"}}`
   → `{:callers [["tools" "k/thread-ids-i"] ["kernel" "thread-ids-i"] ["main"
   "k/thread-ids-i"]], :target "@kernel#1518"}` — nonempty, and INCLUDING the
   intra-kernel caller.)*

This is the only point at which text is parsed for the graph arm. Every subsequent
graph-arm retrieval is a socket round-trip against the materialized claims.

## The metric — a retrieval op, counted non-gameably

**One retrieval op = one primitive retrieval against the substrate**, logged as one
line. Precisely:

- **Text arm:** one `tsearch` (a single search pass over the source — **alternation in
  one regex is still one pass**) = 1 op; one `tread` (a single file or line-range read)
  = 1 op. A re-read of a file already read counts again (a fresh retrieval into the
  context window). **The learned-alias map is given to both arms at setup (warm-session
  frame), so alias-discovery is NOT a per-task op** — it is amortized once across the
  suite, symmetric with the graph's amortized setup. A backtrack (grep over-matched →
  narrower grep) is 2 ops, but note the *one-directional* substring trap above: the
  minimal-correct path picks the distinctive spelling and avoids the backtrack.
- **Graph arm:** one daemon round-trip (`:callers`, `:query`, `:status`) = 1 op.

**Non-gameability — the four guards:**

1. **One wrapper invocation = exactly one retrieval = one log line.** The wrappers
   cannot batch: `tsearch` runs a single regex (alternation allowed — one pass),
   `tread` reads a single file/range, `gquery` does a single socket round-trip. The
   metric is `wc -l <arm>.oplog`; it equals the retrieval count by construction.
2. **Compound-command gaming is blocked.** A subject cramming ten *separate* searches
   into one shell line does NOT score one op: the text arm has *no raw shell* — only
   `tsearch`/`tread`. (The distinction must-fix #2 turns on: a **single alternation
   regex is one search pass and is allowed**; **ten chained greps `grep a; grep b; …`
   are ten retrievals** and the arm cannot express them in one wrapper call. Anti-gaming
   targets compound *commands*, not single-pass alternation.)
3. **`grep -c` / `grep -l` still counts as the search it is** — one search pass = 1 op;
   it sees a count/file-list, not the structure.
4. **No oracle leakage.** Neither arm's CLI exposes the ground-truth scorer. The graph
   arm calls the *same daemon a deployed agent would*, over the wire; the text arm greps
   the *same source a deployed agent would*. Neither is handed the resolved answer.

Wall-clock is recorded but **only as a disclosed secondary measurement, never the
headline** (attributed to model/JVM noise + serialization, per the sibling's M6).

## Owned-resolution validity — the four guarantees (the experiment-invalidating trap)

The audit (`../owned-resolution-audit/REPORT.md`, Q4) scored the prior design
`graph_arm_valid = false` because it listed a text-re-parse CLI (`:resolver-callgraph`)
inside the graph substrate and routed tasks (c)/(d) through `:query`, which cannot serve
them (the warm store materializes `refers_to` but not a `calls-defn` edge, and `:query`
filters `refers_to` out of its read view — `cnf_coord_daemon.clj:50,174`; **verified:
`{:op :reaches}` and `{:op :type-refs}` both return `{:error "unknown op"}`**). This
design enforces all four required guarantees, **and each is verified**:

- **(i) The daemon is populated with the corpus AS CLAIMS, once, at setup** — never per
  query. The temp flat log holds the projected, module-namespaced AST (`kind/v/fN`) of
  all 11 modules (see *The setup*), so `resolve-warm-store!` materializes `refers_to`
  over real code. We do **not** copy the sibling's synthetic-proxy + per-op re-parse
  pattern (`concurrent-authoring/graph_arm.sh:370-381` seeds ~4 proxy claims and
  re-projects text per op — the INVALID shape). **Verified: 101,671 corpus claims in the
  warm store, not a proxy handful.**

- **(ii) Every graph-arm retrieval is a daemon socket round-trip (`gquery`) and nothing
  else.** The `gquery` wrapper speaks only the daemon wire protocol; it cannot shell
  `racket --emit-edn` or `resolve.clj callgraph`. **`:resolver-callgraph` is REMOVED
  from the graph substrate in `tasks.edn`.** The resolver/callgraph CLI is used ONLY to
  compute ground truth once (the oracle), NEVER as a per-query graph-arm retrieval.

- **(iii) The LIVE graph-arm headline is restricted to what the warm store honestly
  answers off resolution-produced claims — tasks (a) and (b)** — because `:callers`
  reads live materialized `refers_to` with `ultimate` chasing (`callers-of-in-store`,

  `[module, spelling]` groups, one identity. Tasks **(c) reaches-closure** and **(d)
  type-refs** are presented in the DETERMINISTIC layer (Layer A) **only**, clearly
  labeled, with their graph cost a **disclosed HYPOTHETICAL** (the op the principled

  blended into a like-for-like ladder, and **never** routed through the cold callgraph
  CLI as a graph-arm retrieval (that is the exact INVALIDATING move; FORBIDDEN).
  Confound must-fix #6: c/d graph cost is labeled "graph cost HYPOTHETICAL (unbuilt op),
  text cost MEASURED."
  - **The principled path to make (c)/(d) live (documented, not faked):** extend

- **(iv) Boot guard (fail-loud) in `reproduce.sh`.** Before ANY measurement, assert:
  1. `{:op :status}` reports a **nonzero** corpus claim count AND its `:log` field is

## The two layers of evidence — what each one actually proves

### Layer A — deterministic structural-cost (a model-INDEPENDENT PROOF, not a measurement)

For each task, a fully scripted comparison: the single graph query vs the
**minimal-correct text reconstruction** (the justified, fair retrieval sequence a
competent ripgrep-wielding engineer MUST perform, scripted as `tsearch`/`tread` calls,
each tied to a *measured* obstacle, **with alternation allowed and the alias amortized**).
The script is identical every run; the number is stable.

**What Layer A proves and what it does NOT.** Layer A removes the agent-model confound
*by removing the agent* — so it is a **structural lower-bound proof about this corpus**
("here is the minimal text reconstruction; it cannot be done in fewer ops"), NOT a claim
about what an agent does. The op-counts in Layer A are **authored** (justified op-by-op),
not measured agent behavior. We say so plainly. Layer A is the model-free **floor** and
**the empirical result of this experiment**; Layer B (which would carry real agent
behavior) is DESIGNED, NOT RUN (see *Layer B* below). (Confound must-fix #1.)

- **Graph cost.** For (a)/(b): **1 live `gquery :callers`** (measured against the running
  daemon). For (c)/(d): the **1 retrieval the principled extension WOULD serve**, priced
  against the once-run callgraph/type-refs oracle and **labeled HYPOTHETICAL, visually
  segregated** from the live rows — until `:reaches`/`:type-refs` land.
- **Text cost.** The minimal-correct path, each op tied to a measured obstacle (spelling
  fan-out caught by alternation; the distinctive-suffix grep that needs no backtrack; one
  convergence read; per-hop frontier expansion for the closure). For (c) the harness
  reports the **concrete scripted-BFS count for THIS corpus** — not an open-ended floor
  (must-fix #5: "≥12" is unfalsifiable; replaced by the measured number).

### Layer B — paired-agent behavioral demo (DESIGNED, NOT RUN)

**Status: this is a design only. Layer B has NOT been run, `reproduce.sh` does NOT
spawn/ingest/validate it, and the static artifacts under `.behavioral/` are NOT a
measured Layer-B result.** The empirical result of this experiment is **Layer A**. The
protocol below specifies what running Layer B *would* require; it remains future work.

The intended design: on the live tasks — (a) direct-callers and (b)
ultimate-through-aliases — spawn **real** text-arm and graph-arm sub-agents. (Task (d)
would run in Layer B as a text-arm accuracy demo on source vs lowered; its graph cost is
the oracle hypothetical, per guarantee iii — never presented as a measured graph op.)

- Each sub-agent would get the **identical prompt**, the **shared learned-alias map**,
  the **same substrate-blind stopping rule** ("stop when you can name the full answer
  set"), and **only its logged retrieval CLI** (text: `tsearch`/`tread`; graph:
  `gquery`). No other tools — critically **no Bash, no Read, no Grep** (must-fix #8: the
  sub-agents must be structurally confined to the wrappers, so "no raw shell" actually
  holds).
- The harness would capture the **actual** op-count (`wc -l` of the arm's oplog) and
  **whether the agent reached the correct answer** (scored against ground truth, below).
- **N ≥ 10 per task per arm** (not 3 — must-fix #1: N=3 cannot separate substrate-signal
  from agent-variance). The report would be the **distribution** (min / median / max
  op-count and the correct-answer rate), not a single run, with run-to-run variance
  disclosed.
- **Wrapper-bypass detector (must-fix #8) — NOT YET IMPLEMENTED.** The design calls for
  `reproduce.sh` to reject any transcript whose observed retrieval count ≠ `wc -l
  <arm>.oplog`, and whose tool-use shows any call outside the exposed wrapper set, so a
  bypassed transcript is **discarded, not scored**. This detector does not exist today;
  it is a precondition for Layer B op-counts to be trustworthy, and must be built before
  Layer B is run.

When run, Layer B could land *above* Layer A (the agent flails — more ops than minimal)
or *below* in op-count only if it skipped required work (which the correctness scorer
would catch). The *minimal-correct* path (Layer A) is the fair model-free floor and is
this experiment's empirical result; Layer B (when run) would report what actually
happens as a distribution — never an authored number dressed as a measurement.

| (e) rename-and-recompile | the **Beagle compiler**: `~/code/beagle/bin/beagle-build-all <dir> --out <o> 2>&1 \| grep -iE "built, .* error"`, parsed for **exactly 0 errors** (never substring `"0 error"`, which passes `built, 10 error(s)`). Clean rename → `11 built, 0 error(s)`; a missed reference fails the build. |
| (cal) trivial-local | direct read of the single file (both arms trivially correct — the conceded tie). |

**Reporting:** per task per arm we report **(op-count, correct?)**. The headline cell is
"same correct answer, N_graph ops vs N_text ops." Where the graph is *also more correct*
— task (d) text-on-lowered (the generic case erases), and task (a) where a qualified-only
text grep silently drops the intra-kernel caller — we report the **accuracy gap too**. An
arm that reaches the WRONG answer cheaply is scored a **loss on that task**, regardless of
op-count.

## The task suite (hop-depth ladder)

Full machine-readable spec in `tasks.edn`. **Live-served by `gquery`:** (cal), (a), (b).
**Layer-A-only (graph cost HYPOTHETICAL, text cost MEASURED — visually segregated):**
(c), (d). **(e)** is the edit-cost corollary, scored by the recompile oracle.

- **(cal) trivial-local** — body of a private helper in one file (`datalog/var?`).
  *Graph:* 1 query. *Text:* 1 `tsearch` + 1 `tread`. **TIE — CONCEDED.**
- **(a) direct callers (1 hop)** — direct callers of `thread-ids-i`. *Graph:* **1**
  `gquery :callers` (live, verified `{main,tools,kernel}`). *Text:* **2** — one
  `tsearch thread-ids-i` (distinctive suffix catches all, no backtrack) + one read of
  the `-n` output to drop def/comment/self-name and keep the real call sites. (Alias
  already known — warm session.) **GRAPH 1 vs TEXT 2.** The *wrong-cheap* path
  (`k/thread-ids-i` only) is also WRONG.
- **(b) ultimate through alias/spelling chains** — given the reference `k/->Claim` at
  its site, find the one binding it denotes and all references across the three
  spellings. *Graph:* **1** `gquery :callers` keyed on the **reference's ultimate target
  node** (verified live, 10 distinct spelling-groups, one identity). *Text:* **2** — one
  alternation `tsearch '(k/)?(->)?Claim\b'` (catches all three spellings in one pass,
  verified 74 tokens) + one read at `kernel.bclj:101` to confirm convergence on the
  single `defrecord`. **GRAPH 1 vs TEXT 2.**
- **(c) transitive blast radius (k hops)** — full reaches-closure of
  `kernel#thread-ids-i`. *Graph (Layer A, HYPOTHETICAL; live once `:reaches` lands):* 1
  retrieval — the Datalog fixpoint returns the whole **7-member** closure. *Text:* the
  scripted BFS — a search per frontier node, per hop, with spelling disambiguation —
  reporting the **concrete measured count for this corpus**. **GRAPH 1 (hypothetical)
  vs TEXT [measured].** This is where the gap is largest, and the graph "1" is labeled
  hypothetical.
- **(d) type-reference reasoning** — which functions take `(Vec Claim)`? The erased-
  *generic* case. *Graph (Layer A, HYPOTHETICAL; live once `:type-refs` lands):* 1
  retrieval — resolved type refs. *Text-on-source:* **3** — one alternation
  `tsearch '\(Vec (k/)?Claim\)'` (both spellings, one pass) + one read to confirm the
  inner `Claim` is the kernel one (alias known). *Text-on-lowered:* **WRONG for the
  generic case** — `(Vec Claim)` erases to 0; the headline COST comparison is
  graph-1-hypothetical vs **text-on-source 3, both correct** (must-fix #7: keep (d) a
  COST comparison on SOURCE). The lowered-blind sub-case is presented ONLY with the
  explicit precondition that **source is unavailable** (the deployed-tooling-reads-
  lowered scenario, matching tier-1's concession), and tightened to: text-on-lowered
  **cannot recover the `(Vec Claim)` generic-parameter fns** (it CAN still see the 4
  surviving single-param `^Claim` hints at `kernel.clj:38,70,74,77` — verified — so
  "structurally blind to Claim" is an overclaim and is dropped).
- **(e) edit-propagation corollary** — rename `Claim → Datum` and make it recompile.
  *Graph:* one identity edit (`resolve.clj rename`, supersedes ONE binding-name claim —
  verified `CLAIMS EDITED: 1`, `resolve.clj:1075`; every `refers_to` re-points) + one
  recompile verify. *Text:* the strongest single `sed -E 's/\bClaim\b/Datum/g'` over
  source + recompile-verify + fix any miss the compiler surfaces. **CONCEDED: a perfect
  single sed over source can match the rename in isolation** (the sibling proved this).
  The graph's structural win is identity O(1) + zero-miss-by-construction; scored by the
  recompile oracle (exactly 0 errors). Reported as **edit-cost, separate from the
  (a)-(d) read-cost headline** (the rename is a graph mutation at setup/oracle time, not
  a per-query retrieval).

## Scope — what this does and does NOT show

**Shows:** for STRUCTURAL / cross-module / transitive questions, an agent reaches the
same verified-correct answer with fewer retrieval operations through the live graph than
by reconstructing structure from text — small at 1 hop (1 vs 2, verified live and proven
minimal), growing with hop-depth, and on the erased-*generic*-type case the graph is
cheaper while text-on-lowered cannot answer at all. The LIVE graph arm (a, b) is
**proven end-to-end** to query resolution-produced warm `refers_to` (daemon boots over
the projected corpus; `:callers` returns the verified sets). (c)/(d) are priced in the
deterministic layer with a **HYPOTHETICAL** graph cost, clearly segregated, with the
principled daemon extension documented as the path to make them live and measured.

**Concedes (honestly, in both directions):**
- **Trivial-local parity** (task `cal`) — a tie.
- **The graph pre-pays** the projection + resolution + maintenance once (S1/S3),
  **measured at ~37s** here. **The per-query advantage holds only in a WARM session
  (the explicit precondition);** in a cold single-question session the graph loses. The
  text arm's alias-learning is amortized the same way — symmetric framing.
- **The text baseline is the FAIR minimal-correct path** with alternation and a known
  alias — re-derived *down* (a: 4→2, b: 5→2, d-source: 4→3) so a ripgrep-skilled
  engineer cannot beat it. The thesis survives the smaller, honest gap.
- **Tasks (c)/(d) are not live-served today** — `:reaches`/`:type-refs` are `unknown
  op` (verified), the warm store has no `calls-defn` edge, `:query` filters `refers_to`.
  Their graph cost is a labeled HYPOTHETICAL, not a measured op.
- **An LSP on the *source* language** sees value-level callers fine (tier-1's
  concession). The categorical text loss is specifically on the `(Vec Claim)` **generic**
  references erased at lowering (task d, source-unavailable sub-case) and on **cost** for
  the cross-module / transitive cases — not a claim that text tooling is broken.

**Does NOT show / out of scope:**
- Not a wall-clock / throughput result (recorded, never headlined).
- Not concurrency safety — that is the sibling experiment's axis.
- Not a claim the graph understands types or semantics; correctness is delegated to the
  resolver (ground truth) and the compiler (oracle).
- Op-count magnitudes are illustrative of *this* corpus; the **phenomenon** (one query
  vs structure-rebuilt-from-text, advantage monotone in hop-depth) is the result, not
  any single number.

## Reproduce

```
bash reproduce.sh
```

Needs `racket`, `babashka` (`bb`), Clojure (JVM, for the daemon), and Beagle
(`BEAGLE=~/code/beagle`, `BEAGLE_HOME=~/code/beagle`). Non-destructive: it operates on
`mktemp` copies of `src/fram` and a temp coordinator log on a **verified-free** high port
(9200–9990; `ss -ltn` empty before boot, retried on TOCTOU collision), runs the **boot
guard** (`:status` claim-count nonzero + `:log` == our temp log; `:callers` of
`kernel#thread-ids-i` nonempty) before trusting any result, and `trap`-kills every daemon
it boots. It **never** touches port 7977 or `~/.local/state/lodestar/claims.log` (the live
lodestar coordinator). **Layer A** (the deterministic per-task op-count table) runs
model-free, **is regenerated by `reproduce.sh`** (graph ops off the live `gq` oplog, the
once-computed `ground_truth.json`, the text-min op-counts, and the (e) rename+recompile),
and **is the empirical result** of this experiment. You will watch, per task, the graph's
single query and the text arm's reconstruction sequence both reach the same answer scored
against the resolver / compiler — the cost difference firing in front of you.

**Layer B** (paired real sub-agents, N≥10 each, wrapper-confined, bypass-detected) is
**DESIGNED, NOT RUN** — `reproduce.sh` does NOT spawn, ingest, or validate it, and the
static artifacts under `.behavioral/` are captured by hand, not regenerated here. Do not
treat `.behavioral/` as a reproducible Layer-B measurement. Running Layer B per the design
above is future work.
